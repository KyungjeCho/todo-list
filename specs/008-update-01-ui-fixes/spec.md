# Feature Specification: 업데이트 묶음 01 — 로그인/온보딩 비주얼, 타임존 표시, 계획알림 아이콘, 이월 데이터 보존

**Feature Branch**: `008-update-01-ui-fixes`
**Created**: 2026-04-15
**Status**: Draft
**Input**: User description (from `docs/specs/update-feature-01.md`):

```
- ui : 로그인 페이지, 온보딩 페이지 배경 그라데이션 추가 및 각 OAuth 브랜드 아이콘 이미지 추가
- ui : 타임존 아시아/서울, 아시아/도쿄 를 국가명/서울 국가명/도쿄로 변경
- fix : '계획알림' 비활성화 시 icon 변경되지 않음

```

이 묶음은 서로 독립적인 4건의 개선(UI 폴리시 2건, 알림 아이콘 버그 1건, 이월 데이터 보존 동작 변경 1건)을 하나의 스펙으로 묶어 추적한다. 각 항목은 독립적으로 배포/검증 가능하다.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 이월 시 어제 할 일 목록 보존 (Priority: P1)

자정이 지나 오늘로 날짜가 바뀌면, 어제 완료하지 못한 할 일은 오늘로 이월되어 계속 처리할 수 있다. 동시에 "어제" 화면을 열어 보면 어제의 할 일 목록이 **삭제되지 않고 그대로** 보존되어 있어, 사용자는 어제 어떤 일이 있었고 무엇을 마쳤는지/마치지 못했는지를 회고할 수 있다.

**Why this priority**: 사용자가 만들었던 데이터가 사라지는 문제이므로 신뢰성·회고 관점에서 가장 영향이 크다. 시각 폴리시나 아이콘 동기화보다 우선한다.

**Independent Test**: 어제 날짜에 완료/미완료 todo 여러 개를 만들어 두고 자정 경과 시뮬레이션(또는 실제 날짜 전환) 후 (1) 오늘 화면에 미완료 항목이 이월되어 나타나는지, (2) 어제 화면을 열었을 때 어제의 원본 todo들이 그대로 남아 있는지를 각각 독립 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 어제 날짜에 미완료 todo 3개와 완료 todo 2개가 있다, **When** 날짜가 오늘로 전환된다, **Then** 오늘 화면에 미완료 3개가 이월되어 보이고, 어제 화면을 조회하면 원본 5개(미완료 3 + 완료 2)가 그대로 남아 있다.
2. **Given** 어제의 todo가 그대로 보존된 상태, **When** 사용자가 어제 화면에서 해당 항목을 보거나 상세를 연다, **Then** 읽기/회고가 가능하다(편집/체크 변경 가능 여부는 기존 정책을 따른다).
3. **Given** 이미 오늘로 이월된 항목이 존재한다, **When** 이월 루틴이 다시 실행된다, **Then** 동일 항목이 중복 이월되지 않는다.

---

### User Story 2 - '계획알림' 비활성화 상태가 아이콘에 즉시 반영 (Priority: P2)

사용자가 설정에서 '계획알림'을 켜고 끌 때, 화면의 알림 아이콘이 현재 활성/비활성 상태를 시각적으로 즉시 보여준다. 토글을 껐는데도 아이콘이 "켜짐" 모양 그대로라면 사용자는 자신의 알림 상태를 오해하게 된다.

**Why this priority**: 기능적 결함이며, 사용자가 알림 상태를 잘못 인식해 실제 알림을 받지 못하거나 과도한 알림을 받고 있다고 오인할 수 있다.

**Independent Test**: '계획알림' 토글을 여러 번 켜고 끄면서, 매 전환마다 해당 아이콘의 표시 상태가 실제 설정값과 일치하는지를 단독으로 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** '계획알림'이 활성화된 상태, **When** 사용자가 해당 설정을 비활성화한다, **Then** 관련 아이콘이 "비활성" 시각(예: 빗금/꺼짐 표시)으로 즉시 갱신된다.
2. **Given** '계획알림'이 비활성화된 상태, **When** 사용자가 해당 설정을 다시 활성화한다, **Then** 아이콘이 "활성" 시각으로 즉시 갱신된다.
3. **Given** 앱을 재시작했다, **When** 사용자가 계획알림이 꺼져 있던 화면을 연다, **Then** 아이콘은 마지막 저장된 비활성 상태를 반영해 표시된다.

---

### User Story 3 - 로그인/온보딩 화면의 비주얼 업그레이드 (Priority: P3)

신규 사용자가 앱을 처음 열었을 때 마주하는 로그인 화면과 온보딩 화면이 단색 배경 대신 **브랜드 감성의 그라데이션**으로 나타나고, OAuth 제공자 버튼에는 각 제공자의 **공식 브랜드 아이콘 이미지**가 함께 표시되어 어떤 계정으로 로그인하는지 직관적으로 알 수 있다.

**Why this priority**: 기능 결함이 아닌 첫인상·신뢰도에 기여하는 시각적 개선이므로 데이터/기능 이슈보다 우선순위가 낮다. 다만 신규 유입 전환율에 영향을 줄 수 있다.

**Independent Test**: 로그아웃 상태에서 앱을 실행해 로그인 화면과 온보딩 화면을 열었을 때 (1) 배경이 그라데이션으로 렌더링되는지, (2) 각 OAuth 버튼에 해당 제공자 아이콘이 표시되는지 각 화면별로 단독 확인한다.

**Acceptance Scenarios**:

1. **Given** 사용자가 로그아웃 상태로 앱을 연다, **When** 로그인 화면이 표시된다, **Then** 배경이 지정된 그라데이션으로 렌더링되고, 각 OAuth 버튼 옆/안에 해당 제공자 브랜드 아이콘이 표시된다.
2. **Given** 사용자가 온보딩 플로우에 진입했다, **When** 온보딩 화면이 표시된다, **Then** 배경이 동일 계열의 그라데이션으로 렌더링된다.
3. **Given** 시스템 다크 모드 설정, **When** 사용자가 다크 모드 상태로 로그인/온보딩 화면을 연다, **Then** 그라데이션과 브랜드 아이콘의 가독성이 유지된다(대비/대체 에셋 적용).

---

### User Story 4 - 타임존 라벨을 국가명/도시명으로 표시 (Priority: P3)

타임존 선택·표시 UI에서 `Asia/Seoul`, `Asia/Tokyo` 같은 IANA 식별자 대신 **영문 국가명/영문 도시명** 형태(예: `South Korea/Seoul`, `Japan/Tokyo` — 국가명·도시명 모두 앱 언어와 무관하게 영어 고정)로 표시되어, 지리권역(Asia)에 익숙하지 않은 사용자도 자신의 위치를 쉽게 찾고 선택할 수 있다.

**Why this priority**: 가독성/현지화 개선으로 기능 결함은 아니다. 다만 전 세계 타임존을 다루는 화면에서는 체감이 크다.

**Independent Test**: 타임존 선택 화면을 열어 한국·일본 항목이 "국가명/도시명" 형식으로 표시되는지, 검색·정렬이 새 라벨로도 정상 동작하는지 단독 확인한다.

**Acceptance Scenarios**:

1. **Given** 사용자가 타임존 선택 화면을 연다, **When** 한국/일본 타임존 항목을 본다, **Then** 각각 "South Korea/Seoul", "Japan/Tokyo" 형태의 라벨로 표시된다(앱 언어와 무관하게 국가명·도시명 모두 영어 고정).
2. **Given** 사용자가 이미 `Asia/Seoul`로 저장된 상태, **When** 설정 화면을 다시 연다, **Then** 동일 타임존이 "South Korea/Seoul"로 표시되며 저장 값 자체는 기존대로 유지되어 데이터 호환성이 깨지지 않는다.
3. **Given** 사용자가 검색창에 "Korea", "South Korea", "Seoul" 등을 입력한다, **When** 결과가 필터링된다, **Then** 해당 타임존 항목이 결과에 포함된다.

---

### Edge Cases

- **이월 중복 실행**: 네트워크 재시도·기기 시각 보정 등으로 이월 루틴이 같은 날 여러 번 실행되어도 같은 항목이 중복 생성되지 않아야 한다.
- **어제 todo 편집/삭제 정책**: 보존된 어제 todo를 사용자가 직접 편집·삭제하는 것이 허용되는지는 기존 정책을 따른다(본 스펙은 자동 삭제 동작만 제거한다).
- **'계획알림' 토글 실패**: 설정 저장이 실패하면 아이콘 상태도 이전 값으로 롤백되어 실제 설정과 일관되어야 한다.
- **OAuth 아이콘 에셋 누락/로딩 실패**: 브랜드 아이콘 이미지를 불러오지 못하더라도 로그인 버튼 자체는 제공자 이름과 함께 동작해야 한다.
- **다크 모드/저전력 모드**: 그라데이션이 가독성(텍스트 대비)을 해치지 않아야 한다.
- **매핑되지 않은 IANA 타임존**: `TZ_TO_COUNTRY_CITY` 매핑에 없는 IANA 식별자는 원문 그대로 폴백 표시한다. 국가명·도시명 모두 영어 고정이므로 번역 누락 문제는 원천적으로 발생하지 않는다.
- **과거(그저께 이전) 데이터**: 이번 변경은 "이월 시 어제 삭제" 로직만 제거한다. 더 과거 데이터에 대한 보존 기간은 기존 정책을 유지한다.

## Requirements *(mandatory)*

### Functional Requirements

#### Carryover (이월)

- **FR-001**: 시스템은 날짜 전환(자정 경과) 시 기존 이월 로직을 계속 적용해 어제 미완료 todo를 오늘로 가져와야 한다.
- **FR-002**: 시스템은 이월 과정에서 어제 날짜에 속한 todo(완료·미완료 모두)를 **삭제해서는 안 된다**.
- **FR-003**: 시스템은 동일 항목이 이월 루틴 재실행 시 중복 생성되지 않도록 보장해야 한다.
- **FR-004**: 사용자가 "어제" 화면을 열었을 때 어제의 원본 todo 목록(완료/미완료 상태 포함)이 그대로 조회되어야 한다.

#### '계획알림' 아이콘 상태

- **FR-005**: 시스템은 '계획알림' 설정값의 변경을 관련 아이콘 UI에 즉시 반영해야 한다(활성 ↔ 비활성 시각 구분).
- **FR-006**: 시스템은 앱 재시작·화면 재진입 시에도 '계획알림' 아이콘이 마지막 저장된 설정값과 일치해 표시되도록 해야 한다.
- **FR-007**: '계획알림' 설정 저장이 실패하면 아이콘 표시 역시 이전 값으로 되돌아가야 한다.

#### 로그인/온보딩 비주얼

- **FR-008**: 시스템은 로그인 화면과 온보딩 화면의 배경을 그라데이션으로 렌더링해야 한다.
- **FR-009**: 시스템은 각 OAuth 제공자 버튼에 해당 제공자의 브랜드 아이콘 이미지를 함께 표시해야 한다.
- **FR-010**: 브랜드 아이콘 이미지 로딩이 실패하더라도 OAuth 로그인 버튼 자체의 동작(탭 → 인증 플로우)에는 영향을 주지 않아야 한다.
- **FR-011**: 그라데이션과 아이콘은 다크 모드에서도 텍스트/버튼의 가독성과 접근성(대비)을 유지해야 한다.

#### 타임존 라벨

- **FR-012**: 시스템은 타임존 선택·표시 UI에서 `Asia/Seoul`, `Asia/Tokyo`를 각각 "South Korea/Seoul", "Japan/Tokyo" 형태의 "영문 국가명/영문 도시명" 라벨로 표시해야 한다(다른 지역에 대해서도 동일 규칙 확장은 향후 범위로 분리 가능).
- **FR-013**: 국가명과 도시명은 앱 언어와 무관하게 **영어로 고정**하여 표시한다(i18n 리소스 추가 없음). 매핑이 없는 IANA 식별자는 원문 그대로 폴백한다.
- **FR-014**: 내부 저장 값은 기존 IANA 식별자(`Asia/Seoul` 등)를 그대로 유지하여 기존 사용자·백엔드 데이터와의 호환성이 깨지지 않아야 한다.
- **FR-015**: 타임존 검색 기능은 새 라벨(영문 국가명·영문 도시명) 및 IANA 원문으로 모두 매칭되어야 한다.

### Key Entities *(include if feature involves data)*

- **Todo (이월 대상)**: 날짜 소속(어제/오늘), 완료 상태, 이월 여부 플래그/원본 참조를 갖는다. 본 업데이트로 "어제 소속 원본" 레코드가 이월 이후에도 삭제되지 않고 보존된다.
- **알림 설정(계획알림)**: 활성/비활성 불리언 값. UI 아이콘 상태는 이 값의 단일 소스를 따른다.
- **타임존 라벨**: 내부 식별자(IANA)와 표시 라벨(영문 국가명/영문 도시명)의 매핑. 저장은 식별자, 표시는 라벨.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 자정 경과 후 어제 화면을 조회했을 때 어제의 원본 todo가 보존되어 보이는 비율이 100%이다(자동 삭제로 인한 유실 0건).
- **SC-002**: 동일 미완료 항목이 이월로 인해 오늘 목록에 중복 생성되는 사례가 0건이다(같은 이월 주기 내 기준).
- **SC-003**: '계획알림' 토글 변경 후 아이콘 표시가 실제 설정값과 일치하지 않는 케이스가 0건이며, 설정 변경 후 1초 이내 아이콘이 갱신된다.
- **SC-004**: 신규 사용자가 로그인 화면에서 사용할 OAuth 제공자를 **아이콘만 보고** 식별하는 데 걸리는 시간이 기존 텍스트-only 대비 감소한다(사용성 테스트 5명 기준 평균 응답 시간 단축).
- **SC-005**: 타임존 선택 화면에서 한국·일본 사용자가 자신의 타임존을 찾는 데 걸리는 시간이 기존("Asia/..." 접두) 대비 감소하고, 잘못된 타임존을 선택해 다시 변경하는 비율이 감소한다.
- **SC-006**: 본 업데이트 배포 후 "할 일이 갑자기 사라졌다"는 취지의 문의/리포트가 이전 기간 대비 유의미하게 감소한다.

## Assumptions

- 현재 시스템의 이월 로직은 "어제 화면" 렌더링을 위해 어제 날짜의 원본 todo를 필요로 하지 않는 방식으로 동작하고 있어, 원본 보존만으로 회고 조회가 가능하다(실제 구현 확인 후 필요 시 조정).
- "이월 시 이전 어제의 todo를 삭제"한다는 사용자 설명은 **이월 루틴 실행 시점에 어제 레코드가 제거되는 동작**을 의미한다고 해석한다.
- '계획알림' 비활성화 아이콘은 기존 활성 아이콘과 시각적으로 명확히 구분되는 변형(예: 빗금·off 상태)이 존재하거나 추가 가능하다.
- 로그인/온보딩 그라데이션 색상과 OAuth 브랜드 아이콘 에셋의 정확한 디자인 사양은 디자이너가 제공하거나, 각 OAuth 제공자의 브랜드 가이드라인을 따른다.
- 타임존 국가명·도시명 모두 **영어 고정**(소스 코드 상수)으로 제공되며 언어별 번역 리소스를 요구하지 않는다. 매핑이 없는 IANA는 원문 그대로 폴백.
- 본 업데이트는 DB 스키마 변경이 필요 없는 범위를 기본 가정으로 한다. 이월 보존을 위해 기존 컬럼/쿼리 수정만으로 충분하지 않다면 구현 단계에서 재평가한다.

## Scope Boundaries

**In scope**

- 이월 루틴에서 "어제 삭제" 동작 제거 및 이월 로직 유지
- '계획알림' 아이콘 상태 동기화
- 로그인·온보딩 화면 배경 그라데이션 및 OAuth 브랜드 아이콘 추가
- 한국/일본 타임존 라벨을 "영문 국가명/영문 도시명" 형식으로 변경(i18n 리소스 추가 없음)

**Out of scope**

- 이월 대상·주기·알림 정책 자체의 변경
- '계획알림' 외 다른 알림 종류의 아이콘 재설계
- 로그인·온보딩 화면의 카피/플로우 변경
- 전 세계 모든 타임존에 대한 라벨 규칙 전면 재정의(한/일 외 지역은 후속 범위에서 다룰 수 있음)
