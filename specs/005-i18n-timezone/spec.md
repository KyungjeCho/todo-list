# 기능 명세: 다국어(i18n) 지원 및 전 세계 타임존 확장

**피처 브랜치**: `005-i18n-timezone`
**작성일**: 2026-04-10
**상태**: Draft
**입력**: "다중 언어(ko, en, ja, es) 지원, 디바이스 언어/타임존 자동 감지, 설정에서 언어 변경, 타임존을 전 세계 IANA로 확장"

## 사용자 시나리오 및 테스트 *(필수)*

### 사용자 스토리 1 — 첫 구동 시 자동 언어 감지 (우선순위: P1)

신규 사용자가 앱을 설치하고 처음 실행한다. 앱은 디바이스의 언어 설정을 자동으로 감지하여 해당 언어로 전체 UI 텍스트를 표시한다. 디바이스 언어가 지원 언어(한국어, 영어, 일본어, 스페인어) 중 하나이면 해당 언어로 표시하고, 지원하지 않는 언어이면 영어(기본 언어)로 표시한다.

**우선순위 근거**: i18n 기능 전체의 기반이다. 자동 언어 감지 없이는 사용자가 읽을 수 없는 언어가 표시되어 앱 사용이 불가능하다. 글로벌 접근성의 핵심.

**독립 테스트**: 일본어로 설정된 디바이스에 앱을 설치한다. 별도 설정 없이 첫 실행 시 모든 UI가 일본어로 표시되어야 한다. 비지원 언어(예: 아랍어) 디바이스에서는 영어로 표시되어야 한다.

**인수 시나리오**:

1. **Given** 디바이스 언어가 일본어로 설정된 상태에서, **When** 앱을 처음 실행하면, **Then** 모든 UI 텍스트가 일본어로 표시된다.
2. **Given** 디바이스 언어가 프랑스어(비지원)로 설정된 상태에서, **When** 앱을 처음 실행하면, **Then** 모든 UI 텍스트가 영어(fallback)로 표시된다.
3. **Given** 디바이스 언어가 한국어로 설정된 상태에서, **When** 앱을 처음 실행하면, **Then** 모든 UI 텍스트가 한국어로 표시된다.

### 사용자 스토리 2 — 설정에서 언어 수동 변경 (우선순위: P1)

로그인한 사용자가 설정 화면에 진입하여 타임존 아래에 있는 "언어" 항목을 탭한다. 4개 지원 언어 목록이 각 언어의 원어명(한국어, English, 日本語, Español)으로 표시된다. 새 언어를 선택하면 앱 재시작 없이 전체 UI 텍스트가 즉시 선택한 언어로 전환된다. 변경 사항은 서버에 저장된다.

**우선순위 근거**: 사용자는 자동 감지를 재정의할 수 있어야 한다. 디바이스 언어와 다른 언어를 선호하거나 디바이스를 공유하는 경우가 있다. 핵심 i18n 상호작용이다.

**독립 테스트**: 로그인 후 설정 → 언어 → "Español" 선택. 앱 전체 UI가 즉시 스페인어로 전환되어야 한다. 앱 종료 후 재실행해도 스페인어가 유지되어야 한다.

**인수 시나리오**:

1. **Given** 사용자가 설정 화면에 있는 상태에서, **When** 타임존 아래의 "언어" 항목을 탭하면, **Then** 4개 언어가 원어명(한국어, English, 日本語, Español)으로 표시된다.
2. **Given** 사용자가 언어 목록에서 새 언어를 선택한 상태에서, **When** 선택이 확정되면, **Then** 앱 전체 UI 텍스트가 재시작 없이 즉시 전환된다.
3. **Given** 사용자가 새 언어를 선택한 상태에서, **When** 선택이 확정되면, **Then** 언어 설정이 서버에 저장된다.
4. **Given** 사용자가 언어를 변경했으나 서버 저장이 실패한 상태에서, **When** 에러가 발생하면, **Then** 로컬 UI는 선택한 언어를 유지하고 에러 토스트가 표시된다.
5. **Given** 서버 저장이 실패한 상태에서, **When** 앱을 재시작하면, **Then** 서버에 저장된 이전 언어 값으로 복원된다.

### 사용자 스토리 3 — 회원가입 시 타임존 자동 감지 (우선순위: P1)

신규 사용자가 OAuth를 통해 회원가입한다. 가입 과정에서 앱이 디바이스의 타임존(IANA 형식)을 자동으로 감지하여 감지된 언어와 함께 서버로 전송한다. 온보딩에서 타임존을 수동 선택할 필요가 없으며, 계획 시간과 회고 시간 설정만 진행된다.

**우선순위 근거**: 온보딩 마찰을 제거하고 처음부터 정확한 알림 스케줄링을 보장한다. 타임존 정확도는 핵심 가치(정시 알림)에 직결된다.

**독립 테스트**: America/Chicago 타임존 디바이스에서 회원가입 완료. 서버 프로필에 타임존이 "America/Chicago"로 저장되어 있는지 확인. 온보딩에서 계획/회고 시간 설정만 표시되고 타임존 선택은 없어야 한다.

**인수 시나리오**:

1. **Given** 디바이스 타임존이 "America/New_York"인 상태에서, **When** OAuth 회원가입을 완료하면, **Then** "America/New_York"이 서버로 전송되어 사용자 프로필에 저장된다.
2. **Given** 자동 감지된 타임존으로 가입을 완료한 상태에서, **When** 온보딩에 진입하면, **Then** 계획 시간과 회고 시간 설정 단계만 표시된다(타임존 선택 없음).
3. **Given** 디바이스 타임존을 감지할 수 없는 상태에서, **When** 가입이 완료되면, **Then** UTC가 fallback으로 적용되고 설정에서 수동 변경할 수 있다.
4. **Given** 온보딩 진입 조건에서, **When** 사용자에게 planTime과 reviewTime이 모두 설정되어 있으면, **Then** 타임존 값과 무관하게 온보딩이 생략된다.

### 사용자 스토리 4 — 설정에서 전 세계 타임존 선택 (우선순위: P1)

로그인한 사용자가 설정에서 타임존 항목을 탭한다. 기존 7개 옵션 드롭다운 대신 전용 선택 화면이 열린다. 전 세계 IANA 타임존이 UTC 오프셋 오름차순으로 정렬되어 표시되고, 현재 선택된 타임존은 최상단에 체크마크(✓)와 함께 고정된다. 상단 검색 필드에서 도시명 또는 타임존 이름으로 대소문자 무관 부분 일치 실시간 필터링이 가능하다. 새 타임존을 선택하면 서버에 저장되고 설정 화면으로 돌아간다.

**우선순위 근거**: 타임존 범위를 7개에서 전 세계로 확장하는 것은 글로벌 사용자 지원과 정확한 알림 전달에 필수적이다.

**독립 테스트**: 설정 → 타임존 탭. IANA 타임존 전체 목록이 표시되어야 한다. "Tokyo" 검색 → Asia/Tokyo 표시. 선택하면 설정 화면에 Asia/Tokyo로 갱신되어야 한다.

**인수 시나리오**:

1. **Given** 사용자가 타임존 설정을 탭한 상태에서, **When** 타임존 선택 화면이 열리면, **Then** 전 세계 IANA 타임존이 UTC 오프셋 오름차순으로 정렬되어 표시되고, 각 항목에 타임존 이름과 UTC 오프셋이 함께 표시된다.
2. **Given** 현재 타임존이 "Asia/Seoul"인 상태에서, **When** 타임존 목록이 표시되면, **Then** "Asia/Seoul"이 최상단에 체크마크(✓)와 함께 고정된다.
3. **Given** 상단에 검색 필드가 표시된 상태에서, **When** "new_y"를 입력하면, **Then** "America/New_York"이 필터링 결과에 100ms 이내로 표시된다(대소문자 무관 부분 일치).
4. **Given** 사용자가 검색어를 입력한 상태에서, **When** 일치하는 결과가 없으면, **Then** "검색 결과가 없습니다" 빈 상태 메시지가 표시된다.
5. **Given** 검색 필드에 텍스트가 입력된 상태에서, **When** 초기화(✕) 버튼을 탭하면, **Then** 검색어가 초기화되고 전체 타임존 목록이 다시 표시된다.
6. **Given** 사용자가 새 타임존을 선택한 상태에서, **When** 선택이 확정되면, **Then** 해당 항목에 체크마크가 표시되고, 서버에 저장되고, 설정 화면으로 복귀하며, 알림 스케줄이 새 타임존 기준으로 재설정된다.

### 사용자 스토리 5 — 음성 입력 언어 연동 (우선순위: P2)

앱 언어를 일본어로 설정한 사용자가 음성 입력 기능을 사용한다. 음성 인식(STT) 엔진이 자동으로 일본어(ja-JP)로 인식한다. 음성 텍스트 정리(refine)에 사용되는 LLM 프롬프트도 일본어로 전환된다.

**우선순위 근거**: 완전한 다국어 경험에 중요하지만, 이 기능 없이도 앱 사용은 가능하다(텍스트 직접 입력). P1 스토리의 i18n 기반에 의존한다.

**독립 테스트**: 언어를 스페인어로 설정하고 음성 입력을 열어 스페인어로 말한다. STT가 스페인어를 인식해야 하고, 정리된 할 일 텍스트도 스페인어여야 한다.

**인수 시나리오**:

1. **Given** 사용자 언어가 "es"로 설정된 상태에서, **When** 음성 입력을 열면, **Then** STT 엔진이 "es-ES" 로케일로 음성을 인식한다.
2. **Given** 사용자 언어가 "en"으로 설정된 상태에서, **When** 음성 인식 텍스트가 정리 요청으로 전송되면, **Then** 서버가 영어 프롬프트로 LLM을 호출한다.
3. **Given** 사용자 언어의 STT 로케일이 지원되지 않는 상태에서, **When** 음성 입력을 열면, **Then** STT가 "en-US"로 기본 설정된다.

### 사용자 스토리 6 — 기본 언어 영어 전환 (우선순위: P1)

앱의 기본/fallback 언어가 한국어에서 영어로 변경된다. 선택한 언어의 번역 키가 누락된 경우, 한국어 대신 영어 번역이 표시된다. 이를 통해 비한국어 사용자도 항상 이해할 수 있는 텍스트를 볼 수 있다.

**우선순위 근거**: i18n 체계 전체를 뒷받침하는 기반 변경이다. 영어를 fallback으로 사용하지 않으면, 비지원 언어권 사용자에게 한국어가 노출된다.

**독립 테스트**: 언어를 스페인어로 설정. 스페인어 번역이 누락된 키는 영어로 표시되어야 하며, 한국어가 노출되어서는 안 된다.

**인수 시나리오**:

1. **Given** 일본어 번역 파일에 특정 번역 키가 누락된 상태에서, **When** 해당 키가 요청되면, **Then** 영어 번역이 표시된다.
2. **Given** 영어 번역도 누락된 상태에서, **When** 해당 키가 요청되면, **Then** 키 이름이 사용자에게 노출되지 않는다(graceful 처리).
3. **Given** 비지원 언어 디바이스의 신규 사용자가, **When** 앱을 사용하면, **Then** 모든 텍스트가 영어로 표시된다.

### 사용자 스토리 7 — 회원가입 시 언어·타임존 서버 저장 (우선순위: P1)

신규 사용자가 OAuth 회원가입을 완료하면, 감지된 디바이스 언어와 타임존이 서버로 전송되어 사용자 프로필에 저장된다. 언어는 지원 목록에 대해 검증되며 유효하지 않은 값은 영어로 기본 설정된다. 타임존은 유효한 IANA 타임존으로 검증되며 유효하지 않은 값은 null로 설정된다.

**우선순위 근거**: 첫 상호작용부터 서버에 정확한 사용자 설정을 확보하여 올바른 알림 스케줄링과 언어별 서버 처리(예: LLM 프롬프트)를 가능하게 한다.

**독립 테스트**: 스페인어, America/Buenos_Aires 타임존 디바이스에서 가입 완료. 서버에 language="es", timezone="America/Buenos_Aires"가 저장되었는지 확인.

**인수 시나리오** *(US3과 차이: US3은 클라이언트 감지/전송 + 온보딩 조건, US7은 서버 수신/검증/저장에 초점)*:

1. **Given** 디바이스 언어 "ja", 타임존 "Asia/Tokyo"로 가입하는 상태에서, **When** 가입이 완료되면, **Then** 서버에 language="ja", timezone="Asia/Tokyo"가 저장된다.
2. **Given** 비지원 디바이스 언어로 가입하는 상태에서, **When** 가입이 완료되면, **Then** 서버에 language="en"이 기본값으로 저장된다.
3. **Given** 유효하지 않은 타임존 값으로 가입하는 상태에서, **When** 가입이 완료되면, **Then** 서버에 timezone이 null로 저장된다.

### 엣지 케이스

- 앱 실행 중 디바이스 언어를 변경하면? 앱 내 언어 설정을 유지한다. 디바이스 언어는 첫 실행 시 또는 서버 저장값이 없을 때만 확인한다.
- 기존 사용자의 언어 형식(예: "ko-KR")을 새 형식("ko")으로 마이그레이션하면? 서버 마이그레이션이 기존 값을 자동 변환한다.
- 구형 디바이스에서 전체 IANA 타임존 목록을 동적으로 가져올 수 없으면? 앱에 번들된 정적 IANA 타임존 리스트를 fallback으로 사용한다.
- 타임존 선택 후 서버 저장이 실패하면? 에러 토스트가 표시되고 타임존 선택 화면에 머물러 재시도할 수 있다.
- 언어별 텍스트 길이 차이가 클 때(예: 스페인어가 긴 경우)? UI 컴포넌트가 다양한 텍스트 길이를 오버플로나 레이아웃 깨짐 없이 수용해야 한다.
- 타임존 자동 감지 후 사용자가 다른 지역으로 이동하면? 앱은 타임존을 자동 갱신하지 않는다. 사용자가 설정에서 수동으로 변경해야 한다.

## 요구사항 *(필수)*

### 기능적 요구사항

- **FR-001**: 시스템은 4개 언어를 지원해야 한다: 한국어(ko), 영어(en), 일본어(ja), 스페인어(es).
- **FR-002**: 시스템은 첫 구동 시 디바이스 언어를 감지하여 지원 언어이면 해당 언어로, 비지원 언어이면 영어로 표시해야 한다.
- **FR-003**: 시스템은 설정 화면의 타임존 아래에서 앱 언어를 변경할 수 있게 하며, 앱 재시작 없이 전체 화면에 즉시 반영해야 한다.
- **FR-004**: 시스템은 언어 옵션을 현재 앱 언어와 무관하게 각 언어의 원어명(한국어, English, 日本語, Español)으로 표시해야 한다.
- **FR-005**: 시스템은 모든 번역 누락 및 비지원 언어 사용자에 대해 영어를 기본/fallback 언어로 사용해야 한다.
- **FR-006**: 시스템은 한국어 원본 번역 파일을 기준으로 영어, 일본어, 스페인어 번역을 AI를 통해 생성하고, 다수의 AI 모델로 교차 검수해야 한다.
- **FR-007**: 시스템은 OAuth 회원가입 시 디바이스 타임존(IANA 형식)을 감지하여 서버에 자동 전송해야 한다.
- **FR-008**: 시스템은 온보딩 진입 조건을 "timezone == null"에서 "planTime == null AND reviewTime == null"로 변경하고, 온보딩에서 타임존 선택 단계를 제거해야 한다.
- **FR-009**: 시스템은 전 세계 IANA 타임존 목록이 포함된 전용 타임존 선택 화면을 제공하고, 대소문자 무관 부분 일치 실시간 검색을 지원해야 한다.
- **FR-010**: 시스템은 타임존을 UTC 오프셋 오름차순으로 정렬하고, 현재 선택 항목을 최상단에 체크마크(✓)와 함께 고정해야 한다.
- **FR-011**: 시스템은 음성 인식(STT) 로케일을 사용자 설정 언어와 연동해야 한다 (ko-KR, en-US, ja-JP, es-ES).
- **FR-012**: 시스템은 음성 텍스트 정리(refine) 시 사용자 언어에 맞는 프롬프트를 사용해야 한다.
- **FR-013**: 시스템은 언어 및 타임존 변경을 서버에 저장해야 한다. 서버 저장 실패 시 로컬 변경은 유지하되 에러 알림을 표시하고, 다음 앱 실행 시 서버 값으로 복원해야 한다.
- **FR-014**: 시스템은 기존 사용자의 언어 데이터를 지역 형식(예: "ko-KR")에서 언어 코드 형식(예: "ko")으로 마이그레이션하고, 기본값을 "ko-KR"에서 "en"으로 변경해야 한다.
- **FR-015**: 시스템은 클라이언트와 서버 양쪽에서 언어 값을 지원 목록(ko, en, ja, es)에 대해 검증하고, 비지원 값을 거부해야 한다.

### 주요 엔티티

- **User**: 정규화된 `language` 필드(ko/en/ja/es 형식, 기본값 "en")와 자동 감지된 `timezone` 필드(IANA 형식, 가입 시 설정)로 확장.
- **번역 파일**: 4개 JSON 파일(ko.json, en.json, ja.json, es.json)에 전체 UI 텍스트를 기능/화면 단위 네임스페이스로 구조화.

## 성공 지표 *(필수)*

### 측정 가능한 결과

- **SC-001**: 지원 언어 디바이스 사용자가 별도 설정 없이 첫 실행 시 디바이스 언어로 앱을 볼 수 있다.
- **SC-002**: 설정에서 언어 전환 시 200ms 이내에 앱 전체 화면에 반영되며, 앱 재시작이 불필요하다.
- **SC-003**: 앱의 모든 하드코딩 한국어 텍스트가 번역 키로 교체되어, 컴포넌트 코드에 원시 한국어 텍스트가 남지 않는다.
- **SC-004**: 출시 후 1개월 내 비한국어 사용자 비율이 전체 사용자의 10% 이상이다.
- **SC-005**: 타임존 검색 시 입력 후 100ms 이내에 필터링 결과가 표시된다.
- **SC-006**: 신규 사용자가 온보딩에서 타임존을 수동 설정할 필요 없이 자동으로 설정된다.
- **SC-007**: 번역 관련 CS 문의가 월 5건 미만이다.
- **SC-008**: 4개 지원 언어 모두에서 음성 입력이 해당 언어의 음성을 정확히 인식한다.
- **SC-009**: 4개 번역 파일이 한국어 원본 파일과 100% 키 동일성을 유지한다.

## 가정사항

- 사용자가 언어/타임존 설정을 서버에 저장하기 위한 안정적인 인터넷 연결이 있다. 오프라인 변경은 로컬에 적용되고 연결 복구 시 동기화된다.
- 기존 `PATCH /users/me/settings` API를 확장하여 언어 업데이트를 처리할 수 있다(하위 호환성 유지).
- User 엔티티에 `language` 컬럼이 이미 존재하며, 스키마 생성이 아닌 형식/기본값 변경만 필요하다.
- AI 생성 번역(Claude, Gemini 등)이 최소 2개 AI 모델의 교차 검증 후 프로덕션 품질을 제공한다.
- 전체 IANA 타임존 목록을 대부분의 대상 디바이스에서 가져올 수 있으며, 구형 디바이스는 정적 fallback 리스트로 대응한다.
- 기존 알림 스케줄링 시스템이 사용자 타임존 변경 시 스케줄을 재계산할 수 있다.
- 디바이스 언어/타임존 감지 라이브러리가 현재 모바일 프레임워크 버전과 호환된다.
