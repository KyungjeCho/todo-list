# Feature Specification: 공유 버튼 기능 변경

**Feature Branch**: `003-share-button-update`  
**Created**: 2026-04-03  
**Status**: Draft  
**Input**: User description: "공유 버튼 기능 변경 — 나에게 전송 버튼 삭제, 클립보드 복사 버튼 추가, 클립보드 복사 결과 토스트 위치 변경"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 공유하기 및 클립보드 복사 팝업 (Priority: P1)

사용자가 공유 버튼을 탭하면, 기존의 "나에게 전송" 버튼 대신 "공유하기"와 "클립보드 복사" 두 개의 옵션이 순서대로 표시되는 팝업이 나타난다.

**Why this priority**: 핵심 기능 변경 — 팝업 구성이 변경되지 않으면 나머지 기능이 동작할 수 없다.

**Independent Test**: 공유 버튼을 탭하여 팝업이 열리고, "공유하기"와 "클립보드 복사" 버튼이 순서대로 표시되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 할 일 목록에 1개 이상의 항목이 존재할 때, **When** 사용자가 공유 버튼을 탭하면, **Then** "공유하기", "클립보드 복사" 순서로 두 개의 메뉴 항목이 포함된 팝업이 표시된다.
2. **Given** 팝업이 열린 상태에서, **When** "공유하기" 버튼을 탭하면, **Then** 기기의 네이티브 공유 시트가 열린다.
3. **Given** 팝업이 열린 상태에서, **When** 팝업 외부 영역을 탭하면, **Then** 팝업이 닫힌다.
4. **Given** 공유 버튼이 있는 화면에서, **When** "나에게 전송" 버튼을 찾으면, **Then** 해당 버튼은 존재하지 않는다.

---

### User Story 2 - 클립보드 복사 및 결과 토스트 표시 (Priority: P1)

사용자가 "클립보드 복사" 버튼을 탭하면 할 일 목록이 클립보드에 복사되고, 화면 중단과 하단 사이에 결과 토스트가 잠시 표시된 후 자동으로 사라진다.

**Why this priority**: 사용자가 복사 결과를 직관적으로 확인할 수 있는 핵심 피드백 UX이다.

**Independent Test**: "클립보드 복사"를 탭한 뒤, 클립보드에 데이터가 복사되었는지 확인하고, 토스트가 화면 중하단 영역에 표시되었다가 자동으로 사라지는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 팝업이 열린 상태에서, **When** "클립보드 복사" 버튼을 탭하면, **Then** 할 일 목록이 클립보드에 복사되고 팝업이 닫힌다.
2. **Given** 클립보드 복사가 성공하면, **When** 결과 토스트가 표시될 때, **Then** 토스트는 화면 중단과 하단 사이 영역에 나타난다 (공유 버튼 바로 아래가 아님).
3. **Given** 결과 토스트가 표시되면, **When** 일정 시간(약 2초)이 경과하면, **Then** 토스트가 자동으로 사라진다.
4. **Given** 클립보드 복사가 실패하면, **When** 오류 토스트가 표시될 때, **Then** 오류 토스트도 동일한 화면 중하단 위치에 표시된다.

---

### Edge Cases

- 할 일이 없는 상태에서 공유 버튼을 탭하면 기존과 동일하게 "공유할 할 일이 없습니다" 안내가 표시된다.
- 클립보드 접근 권한이 없거나 복사에 실패할 경우, 오류 토스트가 중하단 위치에 표시된다.
- 토스트가 표시되는 동안 사용자가 다른 동작(스크롤, 탭 전환 등)을 하더라도 토스트가 정상적으로 표시되고 사라진다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 공유 팝업에서 "나에게 전송" 버튼을 제거해야 한다.
- **FR-002**: 공유 팝업에 "클립보드 복사" 버튼을 추가해야 한다.
- **FR-003**: 공유 팝업의 메뉴 항목 순서는 "공유하기" → "클립보드 복사" 순이어야 한다.
- **FR-004**: "클립보드 복사" 버튼 탭 시 현재 할 일 목록을 클립보드에 복사해야 한다.
- **FR-005**: 클립보드 복사 결과 토스트는 화면 중단과 하단 사이 영역에 표시되어야 한다 (기존 공유 버튼 바로 아래 위치가 아님).
- **FR-006**: 결과 토스트는 약 2초 후 자동으로 사라져야 한다.
- **FR-007**: 복사 실패 시 오류 토스트도 동일한 중하단 위치에 표시되어야 한다.
- **FR-008**: 기존 "공유하기" (네이티브 공유 시트) 기능은 변경 없이 유지되어야 한다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 공유 팝업에 "나에게 전송" 버튼이 존재하지 않는다.
- **SC-002**: 공유 팝업에 "공유하기", "클립보드 복사" 순서로 2개의 메뉴 항목이 표시된다.
- **SC-003**: "클립보드 복사" 탭 후 클립보드에 할 일 목록 텍스트가 정상적으로 저장된다.
- **SC-004**: 복사 결과 토스트가 화면 중하단 영역에 표시되며, 약 2초 내에 자동 소멸된다.
- **SC-005**: 기존 "공유하기" 기능(네이티브 공유 시트)이 정상 동작한다.

## Assumptions

- 클립보드 복사 데이터 포맷은 기존 "나에게 전송" 기능과 동일한 텍스트 포맷을 재사용한다.
- 토스트 자동 소멸 시간은 기존과 동일한 약 2초를 유지한다.
- 토스트의 "화면 중단과 하단 사이"는 화면 높이의 약 60~75% 지점을 의미한다.
- 공유 버튼의 위치, 아이콘, 비활성화 조건(할 일 없음) 등 기존 동작은 변경하지 않는다.
