# Feature Specification: 음성 입력 화면 (Voice Input Screen)

**Feature Branch**: `004-voice-input-screen`  
**Created**: 2026-04-04  
**Status**: Draft  
**Input**: User description: "PRD_VOICE_INPUT.md, TECH_SPEC_VOICE_INPUT.md, Voice_Input.png 기반 음성 인식 개선"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 연속 음성 발화로 여러 할 일 한 번에 등록 (Priority: P1)

사용자가 마이크 FAB 버튼을 탭하면 전용 음성 입력 화면으로 이동하여 연속으로 말하며 여러 할 일을 한 번에 등록할 수 있다. 디바이스 내장 음성 인식이 실시간으로 전사 텍스트를 보여주고, 문장이 끝날 때마다 서버가 텍스트를 간결하게 정리하여 임시 할 일 카드로 표시한다. 녹음 종료 시 모든 임시 할 일이 일괄 생성된다.

**Why this priority**: 핵심 가치 제안 — 기존 1회 녹음 = 1개 할 일 제한을 해소하여 빠르게 여러 할 일을 등록할 수 있게 한다.

**Independent Test**: 마이크 FAB 탭 → 음성 입력 화면 진입 → 여러 문장 발화 → 임시 카드 생성 확인 → 종료 버튼 탭 → 할 일 일괄 생성 및 이전 화면 복귀로 전체 흐름을 독립적으로 테스트할 수 있다.

**Acceptance Scenarios**:

1. **Given** 사용자가 Plan 모드의 MainScreen에 있을 때, **When** 마이크 FAB 버튼을 탭하면, **Then** 음성 입력 전용 화면으로 이동하고 자동으로 녹음이 시작된다.
2. **Given** 마이크 권한이 없는 경우, **When** 마이크 FAB 버튼을 탭하면, **Then** 권한 요청 다이얼로그가 표시된다.
3. **Given** 녹음이 진행 중일 때, **When** 사용자가 말을 하면, **Then** 화면 중하단에 실시간으로 전사 텍스트가 표시되고 인식이 진행됨에 따라 업데이트된다.
4. **Given** 사용자가 한 문장을 말하고 잠시 멈출 때, **When** 확정된 전사 텍스트가 서버로 전송되면, **Then** 정리된 텍스트가 화면 상단 임시 할 일 목록에 카드로 추가된다.
5. **Given** 서버 응답 대기 중일 때, **When** 임시 카드가 표시되면, **Then** 원본 텍스트와 함께 로딩 표시가 나타난다.
6. **Given** 녹음이 진행 중이고 임시 할 일이 1개 이상 있을 때, **When** 사용자가 종료 버튼(빨간 네모)을 탭하면, **Then** 녹음이 중단되고 정리 중인 항목 완료를 기다린 후 모든 임시 할 일이 일괄 생성되며 이전 화면으로 복귀한다.
7. **Given** 녹음이 진행 중이고 임시 할 일이 0개일 때, **When** 종료 버튼을 탭하면, **Then** "인식된 할 일이 없습니다" 안내가 표시되고 이전 화면으로 복귀한다.

### User Story 2 - 임시 할 일 삭제 (Priority: P2)

사용자가 음성 인식 중 잘못 인식되거나 불필요한 임시 할 일을 삭제 버튼으로 즉시 제거할 수 있다.

**Why this priority**: 오인식이나 불필요한 항목을 빠르게 정리하여 최종 생성 목록의 정확성을 높이는 보조 기능이다.

**Independent Test**: 임시 할 일이 있는 상태에서 삭제 버튼 탭 → 해당 항목 즉시 제거를 독립적으로 테스트할 수 있다.

**Acceptance Scenarios**:

1. **Given** 임시 할 일 목록에 1개 이상의 항목이 있을 때, **When** 사용자가 해당 항목의 삭제(X) 버튼을 탭하면, **Then** 해당 항목이 확인 없이 즉시 목록에서 제거된다.

---

### User Story 3 - 뒤로가기 취소 처리 (Priority: P3)

사용자가 음성 입력 도중 뒤로가기를 누르면, 임시 할 일이 있는 경우 확인 다이얼로그를 통해 실수로 진행 중인 작업을 잃지 않도록 보호한다.

**Why this priority**: 데이터 손실 방지를 위한 안전장치로, 핵심 기능은 아니지만 사용자 경험에 중요하다.

**Independent Test**: 임시 할 일이 있는 상태에서 뒤로가기 탭 → 확인 다이얼로그 표시 → 삭제/취소 선택에 따른 동작을 독립적으로 테스트할 수 있다.

**Acceptance Scenarios**:

1. **Given** 음성 입력 화면에서 임시 할 일이 1개 이상 있을 때, **When** 뒤로 버튼을 탭하면, **Then** "입력 중인 할 일 N개가 삭제됩니다" 확인 다이얼로그가 표시된다.
2. **Given** 확인 다이얼로그가 표시된 상태에서, **When** [삭제]를 탭하면, **Then** 녹음 중단 후 임시 할 일을 버리고 이전 화면으로 복귀한다.
3. **Given** 확인 다이얼로그가 표시된 상태에서, **When** [취소]를 탭하면, **Then** 화면에 그대로 남는다.
4. **Given** 음성 입력 화면에서 임시 할 일이 0개일 때, **When** 뒤로 버튼을 탭하면, **Then** 확인 없이 녹음 중단 후 즉시 이전 화면으로 복귀한다.

---

### Edge Cases

- 마이크 권한을 거부하면 어떻게 되는가? → "마이크 권한이 필요합니다" 안내 후 이전 화면 복귀
- 음성 인식이 실패(STT 에러)하면 어떻게 되는가? → 해당 발화 무시, 다음 발화 계속 수신
- 텍스트 정리(LLM)가 실패하면 어떻게 되는가? → 원본 전사 텍스트를 그대로 사용하여 카드 표시
- 일괄 생성이 실패하면 어떻게 되는가? → 에러 안내 후 화면 유지, 재시도 가능
- 네트워크가 오프라인이면 어떻게 되는가? → "네트워크 연결을 확인해주세요" 안내 후 녹음 중단, 화면 유지
- iOS에서 60초 음성 인식 제한에 도달하면 어떻게 되는가? → 자동으로 음성 인식 재시작하여 연속 녹음 유지
- 종료 시 마지막 발화의 서버 정리가 아직 진행 중이면 어떻게 되는가? → 정리 완료까지 로딩 상태 표시 후 일괄 생성

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 마이크 FAB 버튼 탭 시 전용 음성 입력 화면으로 이동해야 한다 (기존 직접 녹음 방식 대체)
- **FR-002**: 시스템은 음성 입력 화면 진입 시 디바이스 내장 음성 인식을 자동으로 시작해야 한다
- **FR-003**: 시스템은 음성 인식 중 실시간 전사 텍스트(interim result)를 화면에 표시해야 한다
- **FR-004**: 시스템은 문장이 확정(isFinal)될 때마다 전사 텍스트를 서버에 전송하여 간결하게 정리해야 한다
- **FR-005**: 시스템은 정리된 텍스트를 임시 할 일 카드로 목록에 추가해야 한다
- **FR-006**: 시스템은 서버 정리 대기 중 원본 텍스트와 로딩 표시를 보여주어야 한다
- **FR-007**: 사용자는 임시 할 일 카드의 삭제 버튼으로 개별 항목을 즉시 제거할 수 있어야 한다
- **FR-008**: 시스템은 종료 버튼 탭 시 녹음을 중단하고 모든 임시 할 일을 일괄 생성해야 한다
- **FR-009**: 시스템은 일괄 생성 시 한 세션에서 최대 20개의 할 일을 지원해야 한다
- **FR-010**: 시스템은 텍스트 정리(LLM) 실패 시 원본 전사 텍스트를 그대로 사용해야 한다
- **FR-011**: 시스템은 임시 할 일이 있는 상태에서 뒤로가기 시 확인 다이얼로그를 표시해야 한다
- **FR-012**: 시스템은 기존 오디오 업로드 엔드포인트 호출 시 deprecated 안내 메시지를 반환해야 한다
- **FR-013**: 시스템은 한국어(ko-KR) 음성 인식을 지원해야 한다
- **FR-014**: 시스템은 iOS 60초 음성 인식 제한 시 자동으로 재시작하여 연속 녹음을 유지해야 한다
- **FR-015**: 시스템은 텍스트 정리 요청에 대해 적절한 요청 빈도 제한(rate limit)을 적용해야 한다

### Key Entities

- **임시 할 일 (Draft Todo)**: 음성 입력 세션 중 클라이언트에서 관리되는 임시 항목. 원본 전사 텍스트, 정리된 텍스트, 상태(정리 중/준비됨/에러)를 포함한다.
- **할 일 (Todo)**: 서버에 영구 저장되는 할 일 항목. 내용, 날짜, 상태를 포함하며 일괄 생성으로 한 번에 여러 개가 만들어진다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자가 한 번의 세션에서 평균 3개 이상의 할 일을 생성할 수 있다
- **SC-002**: 문장 정리 결과가 발화 후 2초 이내에 사용자에게 표시된다 (95th percentile)
- **SC-003**: 세션 완료율(종료 버튼으로 할 일 생성 완료)이 70% 이상이다
- **SC-004**: 사용자가 발화 중 실시간으로 인식 텍스트를 확인할 수 있다 (즉각적 피드백)
- **SC-005**: 서버 정리 실패 시에도 원본 텍스트로 할 일 생성이 가능하다 (graceful degradation)

## Assumptions

- 사용자의 디바이스에 한국어 음성 인식 엔진이 설치되어 있다
- 사용자는 안정적인 네트워크 연결 상태에서 기능을 사용한다 (오프라인 모드 미지원)
- 기존 마이크 FAB 버튼의 동작이 직접 녹음에서 화면 네비게이션으로 완전히 대체된다
- 기존 오디오 업로드 엔드포인트는 삭제하지 않고 deprecated 상태로 코드에 보존한다
- 음성 명령(삭제, 수정 등)이나 날짜/우선순위 자동 추출은 범위 외이다
- 다국어 지원은 범위 외이며 한국어(ko-KR) 전용이다
- 한 세션에서 최대 20개의 할 일 생성이 합리적인 상한이다
- 개별 전사 텍스트 길이는 500자, 정리된 할 일 내용은 255자가 상한이다
