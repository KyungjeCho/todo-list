## 1. 목표

### 1.1 배경

현재 앱은 모든 UI 텍스트가 한국어로 하드코딩되어 있어 한국어 사용자만 이용할 수 있다. 또한 타임존 설정이 7개(Asia/Seoul, Asia/Tokyo, America/New_York, America/Los_Angeles, Europe/London, Europe/Paris, UTC)로 제한되어 있어, 해당 지역 외 사용자는 정확한 알림 시간을 받을 수 없다.

글로벌 사용자 확보와 정확한 알림 서비스 제공을 위해, 다국어 지원 체계를 도입하고 타임존 설정을 전 세계로 확장한다.

### 1.2 목표

- **한국어, 영어, 일본어, 스페인어** 4개 언어를 지원하여 글로벌 사용자 접근성을 확보한다.
- 첫 구동 시 **디바이스 언어를 자동 감지**하여 별도의 설정 없이 즉시 사용할 수 있도록 한다.
- 앱의 **기본(fallback) 언어를 영어로 변경**하여 지원하지 않는 언어권 사용자도 자연스럽게 이용할 수 있도록 한다.
- **타임존 설정을 전 세계 IANA 타임존으로 확장**하여 모든 지역의 사용자가 정확한 알림을 받을 수 있도록 한다.
- 회원가입 시 **디바이스 타임존을 자동 감지**하여 서버에 전송하고, 온보딩 마찰을 줄인다.

### 1.3 기존 기능과의 관계

| 항목 | 기존 | 변경 후 |
| --- | --- | --- |
| UI 언어 | 한국어 하드코딩 | 4개 언어 동적 전환 (ko, en, ja, es) |
| 기본 언어 | 한국어 (ko-KR) | 영어 (en) |
| 언어 설정 | User 엔티티에 `language` 컬럼 존재하나 미사용 | 설정 화면에서 변경 가능, 앱 전체에 즉시 반영 |
| 타임존 선택지 | 7개 하드코딩 옵션 | 전 세계 IANA 타임존 (검색 가능) |
| 타임존 초기값 | NULL → 온보딩에서 수동 선택 | 회원가입 시 디바이스 타임존 자동 감지 후 서버 전송 |
| 음성 인식 언어 | ko-KR 고정 | 사용자 언어 설정에 연동 (ko → ko-KR, en → en-US, ja → ja-JP, es → es-ES) |

## 2. 사용자 스토리

### US-I01: 첫 구동 시 자동 언어 감지

> **사용자로서 나는** 앱을 처음 실행했을 때 별도의 설정 없이 내 디바이스 언어로 앱을 사용할 수 있기를 **원한다. 그래서** 추가 설정 없이 즉시 앱을 이해하고 사용할 수 있다.

**Acceptance Scenarios:**

**AC I1.1: 지원 언어 자동 감지**

- **Given** 사용자가 앱을 처음 실행했을 때
- **When** 디바이스 언어가 지원 언어(ko, en, ja, es) 중 하나인 경우
- **Then** 해당 언어로 앱이 표시된다.

**AC I1.2: 비지원 언어 fallback**

- **Given** 사용자가 앱을 처음 실행했을 때
- **When** 디바이스 언어가 지원하지 않는 언어인 경우
- **Then** 기본 언어인 영어(en)로 앱이 표시된다.

**AC I1.3: 언어 설정 서버 동기화**

- **Given** 디바이스 언어 감지가 완료된 상태일 때
- **When** 사용자가 회원가입을 완료하면
- **Then** 감지된 언어가 서버의 사용자 프로필에 저장된다.

### US-I02: 설정에서 언어 변경

> **사용자로서 나는** 설정 화면에서 원하는 언어로 변경할 수 있기를 **원한다. 그래서** 디바이스 언어와 무관하게 내가 편한 언어로 앱을 사용할 수 있다.

**Acceptance Scenarios:**

**AC I2.1: 언어 변경 UI**

- **Given** 사용자가 설정 화면에 진입한 상태일 때
- **When** 타임존 설정 아래의 "언어" 항목을 탭하면
- **Then** 지원 언어 목록(한국어, English, 日本語, Español)이 표시된다.
- **And** 각 언어는 **해당 언어의 원어명**으로 표시되어 어떤 언어 상태에서도 식별할 수 있다.

**AC I2.2: 언어 즉시 반영**

- **Given** 사용자가 언어 목록에서 새로운 언어를 선택했을 때
- **When** 선택이 확정되면
- **Then** 앱 전체 UI 텍스트가 즉시 선택한 언어로 전환된다.
- **And** 서버에 변경된 언어 설정이 저장된다.
- **And** 앱 재시작 없이 즉시 반영된다.

### US-I03: 디바이스 타임존 자동 감지

> **사용자로서 나는** 회원가입 시 타임존이 자동으로 설정되기를 **원한다. 그래서** 별도의 수동 설정 없이 정확한 시간대 기반의 알림을 받을 수 있다.

**Acceptance Scenarios:**

**AC I3.1: 회원가입 시 타임존 자동 전송**

- **Given** 사용자가 OAuth를 통해 회원가입을 진행할 때
- **When** 회원가입이 완료되면
- **Then** 디바이스의 현재 타임존(IANA 형식)이 서버로 전송되어 사용자 프로필에 저장된다.
- **And** 온보딩 화면에서 타임존을 별도로 선택할 필요가 없다.

**AC I3.2: 온보딩 흐름 변경**

- **Given** 신규 사용자의 타임존이 회원가입 시 자동 설정된 상태일 때
- **When** 온보딩 화면에 진입하면
- **Then** 기존과 동일하게 계획 시간과 회고 시간 설정 단계가 진행된다.
- **And** 온보딩 진입 조건이 `timezone == null`에서 `planTime == null AND reviewTime == null`로 변경된다.

### US-I04: 설정에서 전 세계 타임존 선택

> **사용자로서 나는** 설정 화면에서 전 세계 모든 타임존을 검색하고 선택할 수 있기를 **원한다. 그래서** 어느 지역에 있든 정확한 타임존을 설정할 수 있다.

**Acceptance Scenarios:**

**AC I4.1: 타임존 목록 표시**

- **Given** 사용자가 설정 화면에서 "타임존" 항목을 탭했을 때
- **When** 타임존 선택 화면이 열리면
- **Then** 전 세계 IANA 타임존 목록이 UTC 오프셋 오름차순으로 표시된다.
- **And** 현재 선택된 타임존이 목록 최상단에 고정되고 체크마크(✓)로 강조된다.
- **And** 각 항목에 타임존 이름과 UTC 오프셋이 함께 표시된다 (예: "Asia/Seoul", "UTC+09:00").

**AC I4.2: 타임존 검색 — 실시간 필터링**

- **Given** 타임존 선택 화면 상단에 검색 필드가 표시된 상태일 때
- **When** 사용자가 검색어를 입력하면
- **Then** IANA 타임존 이름(예: "Seoul", "New_York")에 대해 대소문자 무관 부분 일치로 실시간 필터링된다.
- **And** 지역명(예: "Asia", "America", "Europe")으로도 검색할 수 있다.
- **And** 필터링은 입력 즉시(100ms 이내) 반영된다.

**AC I4.3: 타임존 검색 — 결과 없음**

- **Given** 사용자가 검색어를 입력한 상태일 때
- **When** 일치하는 타임존이 없으면
- **Then** "검색 결과가 없습니다" 빈 상태 메시지가 표시된다.

**AC I4.4: 타임존 검색 — 검색어 초기화**

- **Given** 검색 필드에 텍스트가 입력된 상태일 때
- **When** 검색 필드 우측의 ✕ 버튼을 탭하면
- **Then** 검색어가 초기화되고 전체 타임존 목록이 다시 표시된다.

**AC I4.5: 타임존 선택 및 즉시 반영**

- **Given** 사용자가 타임존 목록에서 새로운 항목을 탭했을 때
- **When** 선택이 확정되면
- **Then** 선택한 항목에 체크마크(✓)가 표시되고 이전 화면(설정)으로 자동 복귀한다.
- **And** 설정 화면의 타임존 값이 변경된 값으로 갱신된다.
- **And** 서버에 변경된 타임존이 저장된다.
- **And** 이후 알림 스케줄이 새 타임존 기준으로 재설정된다.

## 4. 화면 설계

### 4.1 설정 화면 — 언어 선택

```
┌─────────────────────────────┐
│  ← 뒤로          설정        │  헤더
│                             │
│  ─── 알림 설정 ───           │
│  계획 시간       08:00   >  │
│  회고 시간       22:00   >  │
│                             │
│  ─── 지역 설정 ───           │
│  타임존     Asia/Seoul   >  │
│  언어         한국어      >  │  ← 신규 항목
│                             │
│  ─── 정보 ───               │
│  오픈소스 라이센스        >  │
│  개인정보 처리방침        >  │
│  문의하기                >  │
│                             │
└─────────────────────────────┘
```

### 4.2 언어 선택 드롭다운

```
┌─────────────────────────────┐
│  언어                        │
│  ┌─────────────────────────┐│
│  │  ✓ 한국어                ││
│  │    English               ││
│  │    日本語                 ││
│  │    Español               ││
│  └─────────────────────────┘│
└─────────────────────────────┘
```

- 각 언어는 원어명으로 표시하여 현재 언어 상태와 무관하게 식별 가능
- 현재 선택된 언어에 ✓ 표시

### 4.3 타임존 선택 화면

```
┌─────────────────────────────┐
│  ← 뒤로      타임존 선택      │  헤더
│                             │
│  ┌─────────────────────────┐│
│  │ 🔍 도시 또는 타임존 검색   ││  검색 필드
│  └─────────────────────────┘│
│                             │
│  ✓ Asia/Seoul    (UTC+09:00)│
│    Asia/Tokyo    (UTC+09:00)│
│    Asia/Shanghai (UTC+08:00)│
│    Europe/London (UTC+00:00)│
│    Europe/Paris  (UTC+01:00)│
│    America/New_York (UTC-05)│
│    America/Los_Angeles(-08) │
│    ...                      │
│                             │
└─────────────────────────────┘
```

- 기존 드롭다운 방식에서 전용 선택 화면으로 변경
- 상단 검색 필드로 실시간 필터링
- 각 항목에 UTC 오프셋 표시
- 현재 선택된 타임존에 ✓ 표시

## 5. 기술 요구사항

### 5.1 다국어 체계 (i18n)

| 항목 | 상세 |
| --- | --- |
| **라이브러리** | `i18next` + `react-i18next` + `expo-localization` |
| **지원 언어** | ko (한국어), en (영어), ja (일본어), es (스페인어) |
| **기본 언어** | en (영어) |
| **번역 파일 구조** | `frontend/src/i18n/locales/{lang}.json` |
| **감지 순서** | 1) 서버 저장값 → 2) 디바이스 언어 → 3) 영어 fallback |
| **네임스페이스** | 단일 네임스페이스 (앱 규모상 분리 불필요) |

### 5.2 번역 파일 구조

```
frontend/src/i18n/
├── index.ts              # i18next 초기화 설정
├── locales/
│   ├── ko.json           # 한국어 (원본)
│   ├── en.json           # 영어 (기본/fallback)
│   ├── ja.json           # 일본어
│   └── es.json           # 스페인어
```

### 5.4 번역 생성 및 검수 프로세스

| 단계 | 설명 |
| --- | --- |
| 1. 원본 작성 | 한국어(ko.json) 번역 파일을 먼저 작성 |
| 2. AI 번역 | 원본을 기반으로 AI(Gemini, Claude 등)가 영어, 일본어, 스페인어 번역 생성 |
| 3. 교차 검수 | 생성된 번역을 다른 AI 모델로 검수하여 자연스러운 표현인지, 문맥에 맞는지 확인 |
| 4. 통합 | 검수 완료된 번역 파일을 프로젝트에 반영 |

### 5.5 타임존 확장

| 항목 | 상세 |
| --- | --- |
| **타임존 목록** | `Intl.supportedValuesOf('timeZone')`으로 런타임에 전체 IANA 타임존 획득 |
| **검색** | 클라이언트 사이드 필터링 (도시명, 타임존 이름 부분 일치) |
| **오프셋 표시** | `Intl.DateTimeFormat`으로 UTC 오프셋 동적 계산 |
| **디바이스 감지** | `expo-localization`의 `getCalendars()[0].timeZone` 또는 `Intl.DateTimeFormat().resolvedOptions().timeZone` |

## 6. 에러 처리

| 시나리오 | 동작 |
| --- | --- |
| 디바이스 언어 감지 실패 | 영어(en) fallback 적용 |
| 번역 키 누락 | 영어(en) 번역 텍스트로 fallback. 영어도 누락 시 키 이름 표시 방지 (빈 문자열보다 영어 원본 유지) |
| 언어 변경 서버 저장 실패 | 로컬에 즉시 반영하되, 에러 토스트 표시. 다음 앱 실행 시 서버 값 복원 |
| 타임존 감지 실패 | UTC fallback 적용, 설정 화면에서 수동 변경 유도 |
| 타임존 검색 결과 없음 | "검색 결과가 없습니다" 메시지 표시 |

## 7. 범위 외 (Out of Scope)

- 앱 내 번역 편집/제안 기능 (크라우드소싱 번역)
- RTL(Right-to-Left) 언어 지원 (아랍어, 히브리어 등)
- 5개 이상의 추가 언어 지원 (향후 확장 가능하나 본 버전에서는 4개)
- 서버 사이드 다국어 (에러 메시지 등 — 본 버전에서는 프론트엔드 표시용 텍스트만)
- 날짜/시간 표시 형식의 로케일별 자동 전환 (예: MM/DD vs DD/MM)
- 통화, 숫자 형식의 로케일 적용
