# DDL Ground Rules

Technical reference for all database schema work in the Friends project.

---

## 1. Table Naming Convention

- Prefix: `FRIENDS_` + `UPPER_SNAKE_CASE`

| Good | Bad |
|------|-----|
| `FRIENDS_USER` | `user` |
| `FRIENDS_CHAT_MESSAGE` | `ChatMessage` |
| `FRIENDS_USER_SESSION` | `friends_userSession` |

---

## 2. Column Naming Convention

- All column names use `snake_case`

| Good | Bad |
|------|-----|
| `user_id` | `userId` |
| `created_at` | `createdAt` |
| `avatar_url` | `avatarUrl` |

---

## 3. Column Order Rule (MANDATORY)

All columns MUST appear in this order:

1. `id` — Primary Key
2. FK columns — Foreign Key role columns
3. Audit columns — `created_at`, `created_by`, `updated_at`, `updated_by`, `deleted_at`
4. Status columns
5. Type columns
6. Business columns

---

## 4. Enum Type Naming

- Format: `{TABLE_NAME}_{COLUMN_NAME}`
- Uses the full table name (including `FRIENDS_` prefix)

Examples:
```sql
-- Table: FRIENDS_FRIEND, Column: visibility
CREATE TYPE FRIENDS_FRIEND_VISIBILITY AS ENUM ('PUBLIC', 'PRIVATE', 'MUTUAL');

-- Table: FRIENDS_CHAT_MESSAGE, Column: sender_type
CREATE TYPE FRIENDS_CHAT_MESSAGE_SENDER_TYPE AS ENUM ('USER', 'AI', 'SYSTEM');
```

---

## 5. Primary Key

- All PKs are UUID using PostgreSQL native `UUID` type
- Column name is always `id`
- UUID is generated by the **Server Application** — NOT by DB DEFAULT

```sql
id UUID CONSTRAINT pkx_tableName_id PRIMARY KEY
```

Example:
```sql
id UUID CONSTRAINT pkx_user_id PRIMARY KEY
```

---

## 6. Audit Columns (Required on ALL Tables)

Every table MUST include these five audit columns:

| Column | Type | Nullable | Description |
|--------|------|----------|-------------|
| `created_at` | `TIMESTAMPTZ` | NOT NULL | Creation timestamp |
| `created_by` | `UUID` | NOT NULL | Creator user ID |
| `updated_at` | `TIMESTAMPTZ` | NOT NULL | Last update timestamp |
| `updated_by` | `UUID` | NOT NULL | Last updater user ID |
| `deleted_at` | `TIMESTAMPTZ` | NULL | Soft delete timestamp (NULL = active) |

---

## 7. Self-Reference Policy

On first user creation, the `id`, `created_by`, and `updated_by` values MUST all be set to the same UUID. This bootstraps the audit trail without requiring a pre-existing user record.

```sql
-- Example: Creating the very first user
INSERT INTO FRIENDS_USER (id, created_by, updated_by, ...)
VALUES ('uuid-v4-here', 'uuid-v4-here', 'uuid-v4-here', ...);
```

## 8. Index Naming Convention

Format: `{prefix}_{table}_{columns}`

### Prefixes

| Prefix | Type |
|--------|------|
| `pkx` | Primary Key |
| `idx` | General index |
| `ux` | Unique index |

### Naming Rules

- **Table name**: Remove `FRIENDS_` prefix, convert to camelCase
  - `FRIENDS_USER` → `user`
  - `FRIENDS_USER_SESSION` → `userSession`
  - `FRIENDS_CHAT_MESSAGE` → `chatMessage`
- **Column names**: camelCase concatenation
  - `user_id` → `userId`
  - `user_id, created_at` → `userIdCreatedAt`
- If total index name length exceeds **63 characters**, abbreviate column names only (not prefix or table name)

### Examples

```sql
-- PK
id UUID CONSTRAINT pkx_user_id PRIMARY KEY

-- General index (single column)
CREATE INDEX idx_user_email ON FRIENDS_USER (email);

-- General index (composite)
CREATE INDEX idx_userSession_userId_createdAt ON FRIENDS_USER_SESSION (user_id, created_at);

-- Unique index
CREATE UNIQUE INDEX ux_user_email ON FRIENDS_USER (email);
```

---

## 9. Foreign Key Policy

- **NO DB-level FK constraints**
- FK relationships are managed at the **application/code level only**
- FK role columns MUST have an index for query performance

```sql
-- FK role column: always indexed, never constrained at DB level
some_user_id UUID NOT NULL,

CREATE INDEX idx_tableName_someUserId ON FRIENDS_TABLE_NAME (some_user_id);
```

---

## 10. Table Template

```sql
CREATE TABLE FRIENDS_TABLE_NAME (
    -- 1. id
    id UUID CONSTRAINT pkx_tableName_id PRIMARY KEY,

    -- 2. fk (FK role columns)
    some_user_id UUID NOT NULL,

    -- 3. audit
    created_at TIMESTAMPTZ NOT NULL,
    created_by UUID NOT NULL,
    updated_at TIMESTAMPTZ NOT NULL,
    updated_by UUID NOT NULL,
    deleted_at TIMESTAMPTZ NULL,

    -- 4. status

    -- 5. type

    -- 6. business columns
);

-- Indexes for FK role columns
CREATE INDEX idx_tableName_someUserId ON FRIENDS_TABLE_NAME (some_user_id);
```
