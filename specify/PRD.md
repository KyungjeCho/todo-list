## 1. 목표

### 1.1 배경

기존 Todo 앱들은 단순히 할 일을 나열하고 체크하는 기능에 그치며, 사용자가 하루를 **의식적으로 설계하고 되돌아보는 루틴**을 형성하기 어렵다. Daily Scrum의 "점검 → 실행 → 회고" 흐름에서 착안하여, 개인 일정 관리에 **계획 → 실행 → 회고** 루틴을 적용하고자 한다.

### 1.2 목표

- 사용자에게 **"계획 → 실행 → 회고"** 의 일정한 데일리 루틴을 제공하여, 하루 단위의 생산성 관리 습관을 형성한다.
- 적시 알림을 통해 사용자가 계획과 회고 단계를 놓치지 않도록 유도한다.
- 미완료 일정의 자동 이월, 할일 추가시AI 마이크 인식, 공유를 통해 **일정 관리의 마찰을 최소화**한다.
- 사용자 스스로가 자신의 생산성 패턴을 인지하고 개선할 수 있도록 돕는다.

## 2. 타겟 사용자 (Personas)

### Persona 1: 고등학생 민수 (18세, 고등학생)

| 항목 | 내용 |
| --- | --- |

| **Pain Point** | 공부와 수업, 여러 과제를 해야 하는데 할 일을 까먹음 |
| **기대** | 수업과 공부, 학원, 과제 등을 등록하여 효율적으로 공부하고 싶음 |
| **행동 패턴** | 아침 등교길에 할 일을 체크하고, 하루 끝나면 뭘 했는지 체크 |

| **기대** | 반복되는 수업/과제 일정을 빠르게 등록하고 싶음 |
| **행동 패턴** | 자기 전에 다음 날 계획을 세우고, 하루 끝에 뭘 했는지 체크 |

| 항목 | 내용 |

## 3. 사용자 스토리

### US-01: 아침 계획 세우기 (Plan)

> **사용자로서나는** 매일 아침 오늘 할 일을 빠르게 계획할 수 있기를 **원한다.그래서** 하루를 목적의식을 가지고 시작할 수 있다.
> 

**Acceptance Scenarios:**

**AC 1.1: 시간대별 모드 전환**

- **Given** 사용자가 앱을 실행하면
- **When** 메인 화면이 로딩되면
- **Then** 현재 시간대에 따라 계획(Plan) 모드 또는 회고(Review) 모드가 자동으로 표시된다.
- **And** 사용자가 원할 경우 수동으로 모드를 전환할 수 있다.

**AC 1.2: 미완료 일정 이월 표시**

- **Given** 어제 미완료된 할 일이 1개 이상 존재할 때
- **When** 사용자가 오늘의 계획 화면에 진입하면
- **Then** 미완료 항목이 오늘 목록에 자동으로 표시된다.

**AC 1.3: 새 할 일 추가**

- **Given** 사용자가 계획 화면에서 새로운 할 일을 등록하고 싶을 때
- **When** "+" 버튼을 탭하고 할 일을 입력하면
- **Then** 입력한 항목이 오늘 목록에 즉시 추가된다.

**AC 1.4: 마이크 음성 인식**

- **Given** 사용자가 계획 화면에서 새로운 할 일을 등록하고 싶을 때
- **When "마이크**" 버튼을 탭하고 말을 하면
- **Then** LLM AI가 음성을 다듬어 할일을 오늘 목록에 즉시 추가된다.

**AC 1.5: 할 일 내용 수정 (탭)**

- **Given** 오늘 목록에 등록된 할 일의 내용을 변경하고 싶을 때
- **When** 사용자가 해당 항목을 탭하여 내용을 수정한 뒤 다른 영역을 탭하면
- **Then** 변경된 내용이 자동으로 저장되어 즉시 반영된다.

**AC 1.6: 할 일 비활성화 (더블 탭)**

- **Given** 오늘 목록에 더 이상 수행하지 않을 할 일이 존재할 때
- **When** 사용자가 해당 항목을 더블 탭하면
- **Then** 해당 항목에 취소선이 표시되고 이월 대상에서 제외된다.

**AC 1.7: 할 일 삭제 (길게 누르기 + 드래그)**

- **Given** 오늘 목록에 완전히 제거하고 싶은 할 일이 존재할 때
- **When** 사용자가 해당 항목을 길게 누르면 하단에 휴지통 UI가 표시되고, 항목을 휴지통으로 드래그하면
- **Then** 해당 항목이 목록에서 제거되고 기록에 남지 않는다.

**AC 1.8: 계획 시간 알림**

- **Given** 사용자가 계획 알림 시간을 설정한 상태일 때
- **When** 설정한 시간이 도래하면
- **Then** 푸시 알림이 발송된다.
- **And** 계획 알림 시간이 설정되지 않은(NULL) 경우 알림이 발송되지 않는다.

### US-02: 일정 실행 및 체크 (Do)

> **사용자로서나는** 실행 중인 일정을 쉽게 체크하고 상태를 관리할 수 있기를 **원한다.그래서** 하루 동안의 진행 상황을 실시간으로 파악할 수 있다.
> 

**Acceptance Scenarios:**

**AC 2.1: 할 일 완료 처리**

- **Given** 오늘 목록에 미완료 할 일이 존재할 때
- **When** 사용자가 해당 항목의 체크박스를 탭하면
- **Then** 해당 항목이 "완료" 상태로 변경되고 시각적으로 구분된다.

**AC 2.2: 메모 추가**

- **Given** 사용자가 특정 할 일 항목을 선택한 상태일 때
- **When** 메모 입력 필드에 텍스트를 작성하고 저장하면
- **Then** 해당 항목에 메모가 첨부되어 이후에도 확인할 수 있다.

### US-03: 하루 회고 (Review)

> **사용자로서나는** 하루가 끝날 때 같은 페이지에서 회고 모드로 전환하여 오늘을 돌아보고 짧은 회고를 남길 수 있기를 **원한다.그래서** 자신의 패턴을 인식하고 내일을 더 잘 계획할 수 있다.
> 

**Acceptance Scenarios:**

**AC 3.1: 회고 시간 알림**

- **Given** 사용자가 회고 알림 시간을 설정한 상태일 때
- **When** 설정한 시간이 도래하면
- **Then** 회고를 유도하는 푸시 알림이 발송된다.
- **And** 회고 알림 시간이 설정되지 않은(NULL) 경우 알림이 발송되지 않는다.

**AC 3.2: 회고 모드 진입 시 진행률 및 요약 표시**

- **Given** 오늘 할 일 목록에 완료 항목과 미완료 항목이 존재할 때
- **When** 페이지가 회고(Review) 모드로 전환되면
- **Then** 오늘의 진행률(완료/전체)과 함께 완료된 항목과 미완료된 항목이 분리되어 요약 표시된다.

**AC 3.3: 완료 여부 최종 체크**

- **Given** 회고 모드에서 오늘의 할 일 목록이 표시된 상태일 때
- **When** 사용자가 미처 체크하지 못한 항목의 완료 여부를 변경하면
- **Then** 해당 항목의 상태가 즉시 반영되고 진행률이 재계산된다.

**AC 3.4: 오늘의 일정 완료**

- **Given** 사용자가 회고 모드에서 완료 여부 체크와 항목 정리를 마친 상태일 때
- **When** "오늘의 일정 완료" 버튼을 탭하면
- **Then** 오늘의 일정이 확정되어 기록으로 저장되고, 미완료 항목은 자동 이월 대상이 된다.

### US-04: 미완료 일정 자동 이월

> **사용자로서나는** 어제 완료하지 못한 일정이 자동으로 오늘로 넘어오기를 **원한다.그래서** 중요한 일을 놓치지 않고 이어서 처리할 수 있다.
> 

**Acceptance Scenarios:**

**AC 4.1: 자정 기준 자동 이월**

- **Given** 어제 날짜의 할 일 중 미완료 항목이 존재할 때
- **When** 자정(00:00)이 지나고 사용자가 앱에 접속하면
- **Then** 미완료 항목이 오늘 날짜의 목록에 자동으로 이월되어 표시된다.

### US-05: 캘린더

> **사용자로서나는** 미래 일정까지 확인할 수 있기를 **원한다.그래서** 모든 일정을 한 곳에서 관리하고 앞으로의 계획을 감안해 오늘을 설계할 수 있다.
> 

**Acceptance Scenarios:**

**AC 5.1: 캘린더 페이지에서 일정 확인**

- **Given** 사용자가 캘린더 페이지에 진입하면
- **When** 날짜를 탐색하면
- **Then** 해당 날짜에 등록된 앱 내 할 일이 표시된다.

### US-06: TodoList 공유

> **사용자로서나는** 나의 TodoList을 다른 사람에게 쉽게 공유할 수 있기를 **원한다. 그래서** 나의 하루 일정 마무리에 대한 감정을 느낄수 있다.
> 

**Acceptance Scenarios:**

**AC 6.1: 일정 선택 및 공유**

- **Given** 사용자가 공유할 일정이 존재할 때
- **When** 특정 날짜 또는 기간을 선택하고 "공유" 버튼을 탭하면
- **Then** 공유 대상 목록 최상단에 "나에게 전송" 옵션이 표시되고, 그 아래에 SNS/메신저 등 공유 채널이 나열된다.

### US-07: 온보딩 및 설정

> **사용자로서나는** 처음 로그인 시 계획/회고 시간을 설정하고, 이후에도 자유롭게 변경할 수 있기를 **원한다.그래서** 나의 생활 패턴에 맞는 루틴 알림을 받을 수 있다.
> 

**Acceptance Scenarios:**

**AC 7.1: 최초 로그인 시 루틴 시간 설정**

- **Given** 사용자가 회원가입 후 처음 앱에 로그인하면
- **When** 온보딩 화면이 표시되면
- **Then** 계획 시작 시간과 회고 시간을 설정하는 단계가 진행된다.
- **And** 기본값(계획 08:00, 회고 22:00)이 미리 채워져 있어 사용자가 바로 확인하거나 조정할 수 있다.
- **And** 시간을 설정해야 메인 화면으로 진입할 수 있다.
- **And** 설정 후에도 설정 화면에서 알림을 해제(NULL)할 수 있다.

**AC 7.2: 설정에서 루틴 시간 변경**

- **Given** 사용자가 설정 화면에 진입한 상태일 때
- **When** 계획 시간 또는 회고 시간을 변경하거나 해제하면
- **Then** 변경된 시간이 즉시 반영되어 이후 알림이 새 시간에 발송된다.
- **And** 시간을 해제(NULL)하면 해당 알림이 더 이상 발송되지 않는다.
